üìö Part 1: Understanding Git and GitHub

What is Git?

Think of Git as a super-powered "Track Changes" feature (like in Microsoft Word), but for all your files. It:

  • Keeps a complete history of every change you make
  • Lets you go back to any previous version of your work
  • Helps you understand what changed, when, and why

Important: Git runs on your computer. It's a separate program that both RStudio and VS Code use behind the scenes.

What is GitHub?

GitHub is a website that stores your Git projects in the cloud. Think of it like:

  • Google Drive or Dropbox, but specifically designed for code
  • A social network for programmers where you can share and collaborate
  • A backup system that keeps your work safe online

Why Should You Care?

  • üìÅ Backup: Your work is safely stored in the cloud ‚Äì never lose your homework again!
  • üîÑ Version History: Made a mistake? No problem! Go back to any previous version
  • üë• Collaboration: Work with classmates without emailing files back and forth
  • üíº Professional Skill: Every tech company uses Git ‚Äì it's a must-have skill!
  • üìã Portfolio: Your GitHub profile becomes a portfolio showing your work to future employers

Step 2: Install Git on Your Computer

Remember: Git is NOT the same as RStudio or VS Code. It's a separate program that must be installed first.

ü™ü Windows
üçé Mac

For Windows Users:

  1. Go to git-scm.com/downloads
  2. Click "Windows" ‚Äì the download starts automatically
  3. Find the downloaded file (usually in your Downloads folder) ‚Äì it's called something like Git-2.x.x-64-bit.exe
  4. Double-click to run the installer

  6. Click "Install"
  7. When it's done, click "Finish"

For Mac Users:

  1. Go to git-scm.com/downloads
  2. Click "macOS"
  3. The easiest method is using Homebrew. Open Terminal (press Cmd+Space, type "Terminal", press Enter)
  4. Copy and paste this command, then press Enter: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  5. After Homebrew finishes installing, type this and press Enter: 
    brew install git
Alternative for Mac: You can also download the installer directly from the website and follow the prompts.

‚úÖ Verify Git is Installed

Let's make sure Git installed correctly:

  1. Open a terminal:
    • Windows: Search for "Git Bash" in the Start menu and open it
    • Mac: Open Terminal (Cmd+Space, type "Terminal")
  2. Type this command and press Enter: 
    git --version
  3. You should see something like: git version 2.43.0
If you see a version number, Git is installed correctly! üéâ
If you see "command not found" or an error, try restarting your computer and trying again. If it still doesn't work, reinstall Git.

Step 3: Tell Git Who You Are

IMPORTANT: This step is REQUIRED! Without it, you'll get errors later. Git needs to know your name and email to track who made each change.
  1. Open a terminal (Git Bash on Windows, Terminal on Mac)
  2. Type this command (replace with YOUR name) and press Enter: 
    git config --global user.name "Your Full Name"

    For example: git config --global user.name "Jane Smith"

  3. Type this command (replace with YOUR email) and press Enter: 
    git config --global user.email "your.email@example.com"
    Important: Use the SAME email address you used to create your GitHub account!

Verify Your Identity is Set:

Type these commands to check:

git config --global user.name
git config --global user.email

You should see your name and email displayed back to you.

Git now knows who you are! ‚úÖ

Step 4: Create a Personal Access Token (PAT)

A Personal Access Token is like a special password that allows RStudio and VS Code to talk to your GitHub account. GitHub no longer accepts regular passwords for this.

Think of it like this: Your GitHub password lets YOU into GitHub via the website. A PAT lets your APPS (like RStudio) into GitHub on your behalf.

Generate Your Token:

  1. Log in to GitHub at github.com
  2. Click your profile picture in the top-right corner
  3. Click "Settings" from the dropdown menu
  4. Scroll ALL the way down on the left sidebar and click "Developer settings" (it's at the very bottom!)
  5. Click "Personal access tokens" in the left sidebar
  6. Click "Tokens (classic)"
  7. Click "Generate new token" button, then "Generate new token (classic)"
  8. You might need to enter your GitHub password again
  9. Fill in the token settings:
    • Note: Give it a descriptive name like RStudio-VSCode-Access or My-Laptop-Git
    • Expiration: Choose 90 days (or "Custom" to match your course duration)
    • Select scopes: Check the box next to:
      • ‚úÖ repo (this will auto-select all sub-options)
      • ‚úÖ user:email (scroll down to find this under "user")
  10. Scroll down and click "Generate token"
üö® CRITICAL - COPY YOUR TOKEN RIGHT NOW!

GitHub will show you the token ONLY ONCE. It looks like a long string starting with ghp_.

  1. Click the copy button (üìã two overlapping squares) next to the token
  2. Paste it somewhere safe IMMEDIATELY:
    • A password manager (best option!)
    • A secure note on your phone
    • A text file you'll keep safe (not ideal, but okay temporarily)

If you lose this token, don't panic! You can always generate a new one by repeating these steps.

üîß Part 3: Connecting RStudio to GitHub

üé® RStudio Setup

Step 1: Tell RStudio Where Git Is

  1. Open RStudio
  2. Go to Tools ‚Üí Global Options (on Mac: RStudio ‚Üí Preferences)
  3. Click "Git/SVN" in the left sidebar
  4. Make sure "Enable version control interface for RStudio projects" is checked ‚úÖ
  5. Check that the "Git executable" field shows a path:
    • Windows: Usually C:/Program Files/Git/bin/git.exe
    • Mac: Usually /usr/bin/git or /usr/local/bin/git
    If it's empty, click "Browse" and navigate to find the git file on your computer.
  6. Click "OK"

Step 2: Store Your PAT (Recommended Method)

  1. In the RStudio Console (bottom-left panel), install the required packages: 
    install.packages("usethis")
install.packages("gitcreds")

    Press Enter and wait for them to install (this might take a minute).

  2. Now store your PAT by running: 
    gitcreds::gitcreds_set()
  3. When prompted, paste your Personal Access Token (the one you copied in Step 4) and press Enter
  4. You should see a confirmation message that your credentials have been stored.
Your PAT is now saved! RStudio will use it automatically. ‚úÖ

Step 3: Verify Everything Works

Run this in the Console to check your setup:

library(usethis)
usethis::git_sitrep()

This gives you a "situation report". Look for:

  • ‚úÖ Name and Email show your information
  • ‚úÖ GitHub user shows your GitHub username
  • ‚úÖ Personal access token shows <discovered>

üíª Part 4: Connecting VS Code to GitHub

üîµ VS Code Setup

VS Code has excellent built-in Git support and makes connecting to GitHub very easy!

Step 1: Open VS Code

  1. Download VS Code from code.visualstudio.com if you haven't already
  2. Install and open VS Code

Step 2: Install the GitHub Extension (Recommended)

  1. Click the Extensions icon in the left sidebar (it looks like 4 squares)
  2. Search for "GitHub Pull Requests and Issues"
  3. Click "Install" on the extension published by GitHub
This extension provides much richer GitHub integration including managing pull requests and issues directly within VS Code!

Step 3: Sign In to GitHub

VS Code makes this easy with automatic sign-in:

  1. Look at the bottom-left corner of VS Code ‚Äì you should see a person icon
  2. Click on it and select "Sign in to Sync Settings" or "Turn on GitHub Pull Requests..."
  3. A browser window will open asking you to authorize VS Code
  4. Click "Authorize"
  5. You'll be redirected back to VS Code ‚Äì you're now connected!
Alternative method: If the automatic sign-in doesn't work, VS Code will prompt you for your GitHub username and password when you try to clone or push. Enter your username and use your PAT as the password.

Step 4: Locate the Source Control Panel

The Source Control view is where you'll do all your Git operations:

  1. Click the Source Control icon in the left sidebar (looks like a branching tree, usually 3rd from top)
  2. This is your Git control center in VS Code!
VS Code is now connected to GitHub! üéâ

üìÇ Part 5: Working with Repositories

Option A: Clone an Existing Repository (Most Common for Students)

"Cloning" means downloading a copy of a repository from GitHub to your computer.

First, Get the Repository URL:

  1. Go to the GitHub repository page (your instructor will give you the link)
  2. Click the green "Code" button
  3. Make sure "HTTPS" is selected (not SSH)
  4. Click the copy button (üìã) to copy the URL

The URL looks like: https://github.com/username/repository-name.git

üé® Clone in RStudio

  1. Go to File ‚Üí New Project
  2. Click "Version Control"
  3. Click "Git"
  4. Paste the URL into "Repository URL"
  5. Choose where to save it (e.g., Documents folder)
  6. Click "Create Project"

Look for the "Git" tab in the top-right panel ‚Äì that's your Git control center!

üîµ Clone in VS Code

  1. Press Ctrl+Shift+P (or Cmd+Shift+P on Mac)
  2. Type "Git: Clone" and select it
  3. Paste the URL and press Enter
  4. Choose where to save it
  5. Click "Open" when prompted

The Source Control panel (left sidebar) is your Git control center!

Option B: Create a New Repository

Step 1: Create the Repository on GitHub

  1. Go to github.com and log in
  2. Click the "+" icon in the top-right corner
  3. Select "New repository"
  4. Give it a name (e.g., my-first-project)
  5. Add a description (optional but recommended)
  6. Choose Public or Private
  7. ‚úÖ Check "Add a README file" ‚Äì this makes cloning easier!
  8. Click "Create repository"

Step 2: Clone It to Your Computer

Follow the cloning instructions from Option A above!

üîß Part 7: Troubleshooting Common Issues

‚ùå "Git executable not found"

Problem: Git may not be installed, or your IDE can't find it.

Solution:

  1. Reinstall Git from git-scm.com
  2. Restart your computer
  3. In RStudio: Tools ‚Üí Global Options ‚Üí Git/SVN ‚Üí Browse to find git
  4. In VS Code: It usually finds Git automatically after restart
‚ùå "Authentication failed" or "Invalid credentials"

Problem: Your PAT is missing, expired, or incorrect.

Solution:

  1. Generate a new PAT on GitHub (follow Step 4 again)
  2. In RStudio: Run gitcreds::gitcreds_set() and paste the new token
  3. In VS Code: When prompted, enter your username and use the PAT as password
‚ùå "Repository not found" when cloning

Problem: The URL is wrong, or you don't have access.

Solution:

  • Check that the URL is correct (copy it again from GitHub)
  • Make sure you have access to the repository (it might be private)
  • Verify you're logged into the correct GitHub account
‚ùå "Updates were rejected" when pushing

Problem: Someone else made changes, or you made changes on another computer.

Solution:

  1. Click Pull first to get the latest changes
  2. Then try Push again
‚ùå Can't see the Git tab/panel

Problem: Your project isn't linked to Git.

Solution:

  • RStudio: Make sure you opened an RStudio Project, not just a file. The project must be cloned from GitHub.
  • VS Code: Make sure you opened a folder that contains a Git repository (has a hidden .git folder).
